OIM Biometrics

El objetivo es completar el proyecto de recolección de datos de las personas deportadas de EUA y México y que entran por aduana aeropuerto o aduana Tecún Umán.

Descripción

La recolección de datos se hace por medio de una GUI desarrollada sobre Laravel y MySQL. Un elemento crucial en el desarrollo de este producto es la integración con un sistema de lectura de información biométrica que analiza huellas digitales.

Actualmente el sistema funciona con una sola marca de lectores de huella digital que lee una sola huella a la vez. El sistema terminado deberá poder utilizar varias marcas de lectores y en específico poder utilizar el lector marca Crossmatch modelo Guardian R2 que el cliente provee para pruebas.

Observaciones técnicas del estado actual del backend

Se hizo una revisión general de todos los controladores, modelos, rutas y otros archivos del código fuente

Existen ciertos patrones en los controladores que no deben de ir en los controladores, como la manipulación de los modelos directamente en las funciones de acción de los controladores. Ejemplo, el cambio de contraseña
Los controladores de usuarios deberían estar distinguidos por los roles de cada usuario. La carencia de esto creará en un futuro un sistema poco sostenible
Se rompieron patrones "conventions" de PHP-SQL en el nombre de las tablas y las columnas: eg. stateId o useraccess debería ser state_id y user_access
Se rompieron patrones de nombres de directorios. eg. models en vez de Models
Las rutas web. routes/web.php no están agrupadas y hay patrones repetidos que pueden simplificarse
Hay files en los modelos que no son modelos como: Common.php
Hay poca limpieza y uniformidad en el estilo del código. eg. app/Libraries/EnLetras.php, existen variables con nombres ambiguos que solo el creador conocería, además de que el nombre EnLetras.php debería seguir ciertos estándares de nombramiento.
No hay ningún test. Ni funcional, ni unitario, ni de aceptación. En un futuro este sistema presentará cada vez más problemas según se vayan haciendo cambios o integraciones.
La base de datos no se creó conforme a migraciones, esto representa un riesgo en la reproducción y mantenimiento de la base de datos, así como en la integridad y seguridad de los mismos datos.
Los archivos de rutas no están distinguidos entre rutas web y de API
El nombre de las rutas no sigue el estándar de nombres de rutas. eg. nombre-de-la-ruta
Hay muchos estilos editados directamente en el HTML
Poco uso de pre-procesadores de CSS
Roles hardcodeados en el HTML @if (\App\models\Common::checkPrivilege(7) || Auth::user()->privileges == "all")
No hay Seeds. En un futuro ésto sugiere que se va a estar trabajando en desarrollo con datos de usuarios reales exponiendo su privacidad.

Observaciones técnicas del hardware biométrico

Realicé una investigación de las especificaciones del SDK y controladores del hardware Guardian R2. Crossmatch provee el SDK L Scan Essentials, que funciona para los dispositivos Tenprint y tiene las siguientes limitaciones:

El sistema operativo sobre el que funciona está limitado a Windows® XP, Windows Vista®, Windows 7, Windows 8, Linux® Ubuntu® (12.4, LTS) for all Guardian (with USB Interface), Patrol, Patrol ID and Verifier 320
Esto sugiere que la integración con la aplicación web actual deba realizarse mediante un puente hacia una máquina virtual de windows 8.1 montada sobre una imagen de Azure. Esta implementación tiene un costo de insfraestructura provisto por Azure. Una vez instalada la imagen, el SDK deberá proveer un canal de comunicación para consumir los datos recolectados por el hardware y exponerlos en la aplicación web.
También puede crearse una máquina virtual de Linux, Ubuntu sobre AWS para exponer un cliente e instalar L Scan Essentials para comunicarse posteriormente con el API del software.
Otros dispositivos con las mismas capacidades que los de Crossmatch funcionan sobre Windows o Linux únicamente limitando el desarrollo e integración a estas plataformas.
NOTA: Algunos dispositivos y hardware ya no están mantenidos por Crossmatch. Hay que verificar las versiones: https://www.crossmatch.com/company/end-of-life/
NOTA: La descarga del software requerido para el modelo Guardian R2: L Scan Essentials requiere credenciales de acceso: https://askpersona.zendesk.com/hc/en-us/articles/203501300-Legacy-SDK-Downloads
NOTA: No he podido encontrar el PDF oficial de documentación de L Scan Essentials, el software para utilizar Guardian R2.
